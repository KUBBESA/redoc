Introducción

Este API esta docoumentado en formato OpenAPI desarrollado por el equipo de desarrollo de Crediviva. Para esta documentación, puedes utilizar el api key special-key para probar los filtros de autorización.

Información de Autenticación

Crediviva ofrece la siguiente forma de autenticación:

  • API Key
  • Bearer JWT Token

bearerHttpAuthentication

JWT Token en el header Authorization, el cual concede de la autenticación necesaria para consumir los recursos expuestos para las operaciones descritas en el API que utilicen este método de autenticación.

Security Scheme Type: HTTP
HTTP Authorization Scheme: Bearer

defaultApiKey

API-Key suministrado por Crediviva, el cual concede de la autenticación necesaria para consumir los recursos expuestos para las operaciones descritas en el API que utilicen este método de autenticación.

Security Scheme Type: API Key
Header parameter name: x-api-key
